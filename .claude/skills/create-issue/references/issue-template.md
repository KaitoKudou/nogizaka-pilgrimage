# Issue テンプレート

Step 2 で Issue 本文を生成する際に使用するテンプレート。

---

## テンプレート

```markdown
## 概要
<!-- 何のIssueかを1〜2行で -->

## 背景
<!-- なぜこの対応が必要か。ユーザー課題・バグの経緯・技術的負債等 -->

## 対応内容
<!-- 実装方針の概要（HOW）。詳細な実装はPRに委ねてよい -->

## Acceptance Criteria
- [ ] （動詞）〜されること
- [ ] （動詞）〜しないこと
<!-- エラーケース・非機能要件も含める。ac-guide.md を参照 -->

## 備考
<!-- スコープ外・関連Issue・参考リンク等。なければ省略 -->
```

---

## 出力の原則

- ACは必ず `- [ ]` のチェックボックス形式で書く（code-review スキルが解析しやすいため）
- ACは「〜されること」「〜しないこと」の形で、コードで検証できる粒度にする
- 曖昧な表現（「適切に」「うまく」「正しく」）はACに使わない
- 対応内容が不明な場合は「TBD」と記載し、背景・ACは必ず埋める
