# コミットメッセージのルール

## フォーマット

```
<type>: <概要>
```

## type 一覧

| type | 使いどころ |
|------|------------|
| feat | 新機能の追加 |
| fix | バグ修正 |
| refactor | 振る舞いを変えないコード改善 |
| test | テストの追加・修正 |
| style | フォーマット・命名等（ロジック変更なし） |
| docs | ドキュメント・コメントの変更 |
| chore | ライブラリ更新・ビルド設定・CI等 |
| perf | パフォーマンス改善 |
| revert | コミットの取り消し |

## 概要の書き方

- 日本語で書く
- 30文字以内
- 文末に句点（。）をつけない
- 動詞で始める（「〜を修正」「〜を追加」）
- 過去形にしない（「修正した」ではなく「修正」）

## 良い例・悪い例

| 良い例 | 悪い例 | 問題点 |
|--------|--------|--------|
| `fix: ログイン時のクラッシュを修正` | `fix` | 内容が不明 |
| `feat: ブックマーク機能を追加` | `いろいろ修正` | typeなし・内容不明 |
| `refactor: LoginViewModelの責務を分離` | `refactor: LoginViewModelのリファクタリングをして責務を分離した` | 長すぎ・過去形 |
| `chore: SnapKit を 5.7.1 にアップデート` | `update` | 内容が不明 |

## コミット前チェック

コミット実行前に以下を必ず確認する。

### デバッグコードの混入

```bash
git diff --staged | grep -E "print\(|debugPrint\(|NSLog\("
```

検出された場合はコミットを止め、ユーザーに確認する。
ただしテストコードや意図的なログ出力（`#log` マクロ等）はこの限りでない。

### 意図しないファイルの混入

```bash
git diff --staged --name-only
```

以下のファイルが含まれている場合は意図的かどうか確認する:

| ファイル | 注意点 |
|----------|--------|
| `*.xcodeproj/project.pbxproj` | ファイル追加・削除・設定変更以外での変更は要確認 |
| `Package.resolved` | 依存更新を意図していないのに変更されている場合は要確認 |
| `*.xcscheme` | スキーム変更を意図していない場合は要確認 |
| `.env`, `*.pem`, `*secret*` | 機密情報の混入。絶対にコミットしない |

### TODO / FIXME の残留

```bash
git diff --staged | grep -E "TODO:|FIXME:"
```

検出された場合はユーザーに確認する（意図的な TODO コメントはそのままでよい場合もある）。
