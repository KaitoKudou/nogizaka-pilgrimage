# Commit Skill

差分を分析し、適切な粒度に分割して、ルールに沿ったコミットメッセージで記録する。

---

## Step 1: 差分の取得

```bash
# ステージング済みの差分
git diff --staged

# 未ステージの差分も含めて全体を把握
git diff HEAD
```

差分が取得できない場合（変更なし・git管理外等）はユーザーに確認する。

---

## Step 2: 差分の分析とコミット分割の提案

`references/split-guide.md` を読み、差分を意味のある単位に分割できるか判断する。

- **1コミットにまとめてよい場合** → Step 3 へ
- **複数に分割すべき場合** → 分割案をユーザーに提示して確認を取る

**分割案の提示フォーマット:**

```
以下の2コミットに分割することを提案します:

① fix: ログインAPIのエラーハンドリングを修正
   対象: LoginRepository.swift, LoginError.swift

② test: ログインエラー系のユニットテストを追加
   対象: LoginRepositoryTests.swift

このまま進めますか？変更があれば教えてください。
```

---

## Step 3: コミット前チェック

`references/commit-rules.md` の「コミット前チェック」セクションに従い、以下を確認する。

- デバッグコードの混入
- 意図しないファイルの混入
- TODO / FIXME の残留

問題が見つかった場合はコミットを止め、ユーザーに報告する。

---

## Step 4: コミットメッセージの生成

`references/commit-rules.md` のルールに沿ってメッセージを生成し、ユーザーに提示して確認を取る。

OK であれば Step 5 へ。

---

## Step 5: コミットの実行

ユーザーの承認後に実行する。

```bash
git commit -m "<承認されたメッセージ>"
```

複数コミットに分割する場合は、ファイルを `git add` で分けてから順に実行する:

```bash
git add <ファイル1> <ファイル2>
git commit -m "<1つ目のメッセージ>"

git add <ファイル3>
git commit -m "<2つ目のメッセージ>"
```

---

## 完了後の出力

コミット完了後、以下を実行してユーザーに報告する。

```bash
git log --oneline -n <コミット数>
```

create-pr スキルへの誘導が適切な場合は、PR 作成に進むか確認する。
